# 이메일 발송 구현 계획

## 발송 서비스

- **Resend** 사용 (이미 send-magic-link.js에서 TODO로 명시됨)
- 무료 플랜: 월 3,000건 / 일 100건 — 초기 운영에 충분
- 발신 주소를 내 도메인으로 쓰려면 도메인 소유 + DNS 레코드 추가 필요
  - 도메인 없으면 Resend 제공 공유 도메인(`@resend.dev`)으로 테스트 가능, 프로덕션엔 부적합

## 준비 사항

1. Resend 계정 생성 + API 키 발급
2. 발신 도메인 구매 (예: `alio-letter.com`) + Resend DNS 인증
3. `.env`에 `RESEND_API_KEY` 추가

## 구현 단계

1. `pipeline/filter.py` — 사용자 PostingFilter를 DB 공고에 적용해 대상 공고 추출
2. `pipeline/mailer.py` — 판정 결과를 HTML 이메일로 포맷 + Resend API 발송
3. `pipeline/daily.py` — 신규 공고 수집 → 분석 → 필터 → 판정 → 발송 전체 자동화
4. `main.py` 메뉴에 daily.py 연결 (또는 cron으로 직접 실행)
5. `send-magic-link.js` Resend 연동 완성 (현재 TODO 상태)

## 실행 흐름 (daily.py)

```
① scraper: 신규 공고 수집 + 상세 크롤링
② analyzer: 미분석 공고 Gemini 분석
③ users 테이블에서 활성 사용자 전체 로드
④ filter: 사용자별 PostingFilter → 매칭 공고 추출
⑤ judge: 매칭 공고 트랙별 자격요건·가점 판정
⑥ mailer: 판정 결과 HTML 이메일 생성 → Resend 발송
```
