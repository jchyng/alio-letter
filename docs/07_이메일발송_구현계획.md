# 이메일 발송 구현

> 기준일: 2026-03-16 (구현 완료)

## 발송 서비스

- **Resend** 사용
- 무료 플랜: 월 3,000건 / 일 100건 — 초기 운영에 충분
- 발신 주소를 내 도메인으로 쓰려면 도메인 소유 + DNS 레코드 추가 필요
  - 도메인 없으면 Resend 제공 공유 도메인(`onboarding@resend.dev`)으로 테스트 가능, 프로덕션엔 부적합
- 무료 플랜 한도 모니터링: https://resend.com/overview (월 1회 확인 권장)

## 준비 사항 (사람이 할 일)

1. Resend 계정 생성 + API 키 발급 → https://resend.com/api-keys
2. 발신 도메인 구매 + Resend DNS 인증 (선택)
3. `pipeline/.env`에 `RESEND_API_KEY`, `RESEND_FROM` 추가
4. Cloudflare Workers Secret에 동일 환경변수 등록

## 실행 흐름 (daily.py)

```
① scraper: 신규 공고 수집 + 상세 크롤링
② analyzer: 미분석 공고 Gemini 분석 → posting_tracks 저장
③ users 테이블에서 is_active=1 사용자 전체 로드
④ filter: 사용자별 PostingFilter → 매칭 공고 추출
⑤ judge: 매칭 공고 트랙별 자격요건·가점 판정 → user_judgments 저장
⑥ mailer: sent_at IS NULL 판정 결과 → HTML 이메일 생성 → Resend 발송 → mark_sent
```
