**cron 로그 확인**


- Python traceback 없는지

### 월 1회

**Resend 발송량 확인** (무료: 일 100건, 월 3,000건)

- https://resend.com/overview 접속
- 이번 달 발송 건수 확인
- 한도 80% 이상이면 유료 플랜 검토 (https://resend.com/pricing)

**Cloudflare D1 용량 확인** (무료: 5GB)

- Cloudflare 대시보드 → Workers & Pages → D1 → `alio-letter` DB
- Storage Used 확인
- 90% 이상이면 오래된 공고 정리 검토

**Gemini API 사용량 확인**

- https://aistudio.google.com → 우측 상단 프로필 → API usage
- 또는 Google Cloud Console → APIs & Services → Gemini API

**잡알리오 HTML 구조 변경 여부 확인**

잡알리오 사이트가 업데이트되면 scraper.py의 CSS 셀렉터가 맞지 않아 수집 실패할 수 있음.

```bash
cd /home/pi/workspace/alio-letter/pipeline
source venv/bin/activate
python main.py 1   # 공고 목록 수집 — 수집 건수 0이면 셀렉터 점검 필요
```

## 장애 대응

### 이메일이 발송되지 않음

1. `daily.log` 에서 `[mailer] 발송 실패` 확인
2. RESEND_API_KEY가 유효한지 확인:
   ```bash
   cat pipeline/.env | grep RESEND
   ```
3. Resend 대시보드에서 발송 로그 확인 → https://resend.com/emails
4. 일 100건 한도 초과 여부 확인

### 공고 수집 건수 0건

1. 잡알리오 접속 가능 여부 확인 (https://www.alio.go.kr)
2. `scraper.py` CSS 셀렉터 점검 — 잡알리오 HTML 구조 변경 가능성
3. IP 차단 여부 확인 (과도한 요청 시 발생 가능)

### daily.py 실행 오류

```bash
# 직접 실행해서 에러 확인
```

## 환경변수 목록

### pipeline/.env (Python 파이프라인용)

| 변수 | 설명 | 예시 |
|------|------|------|
| `GEMINI_API_KEY` | Google Gemini API 키 | `AIzaSy...` |
| `RESEND_API_KEY` | Resend API 키 | `re_...` |
| `RESEND_FROM` | 발신 이메일 주소 | `noreply@alio-letter.com` |

### Cloudflare Workers Secret (CF Functions용)

| 변수 | 설명 |
|------|------|
| `GEMINI_API_KEY` | 회원가입 시 스펙 파싱에 사용 |
| `RESEND_API_KEY` | 가입 확인 메일 · 매직링크 발송 |
| `RESEND_FROM` | 발신 이메일 주소 |

등록 방법:
```bash
cd alio-letter-web
npx wrangler pages secret put GEMINI_API_KEY
npx wrangler pages secret put RESEND_API_KEY
npx wrangler pages secret put RESEND_FROM
```
