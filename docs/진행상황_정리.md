# Alio-Letter 진행 상황 정리

> 최종 업데이트: 2026-03-13 — `06_기술스택_아키텍처.md` 추가

---

## 프로젝트 한줄 요약

잡알리오(job.alio.go.kr) 공기업 채용공고를 AI가 매일 분석해서,
내 스펙에 맞는 공고만 이메일로 자동 발송하는 서비스.

---

## 아키텍처 (현재 확정)

```
[사용자] → Cloudflare Pages (HTML)
    ↓ API 요청
[Cloudflare Functions] → Gemini API (스펙/공고 파싱) → MySQL (라즈베리파이)
    ↓
[라즈베리파이 Cron] → 잡알리오 스크래핑 → Gemini PDF 분석 → MySQL 저장
    → 매칭 쿼리 실행 → 이메일 발송 (Resend API)
```

> 이게 안 되면 뒤에 아무것도 테스트 불가능

- [ ] MySQL Docker 컨테이너 실행 (`alio_letter` DB 생성, `alio` 유저)
- [ ] `schema.sql` 실행 → 테이블 생성
- [ ] `cloudflared` 설치 및 Cloudflare Tunnel 설정 (MySQL 포트 3306)
  - 예: `db.yourdomain.com` → 라즈베리파이 MySQL
- [ ] `wrangler.toml`에 실제 값 입력 (DB_HOST, DB_USER, DB_NAME)
- [ ] `wrangler secret put DB_PASSWORD`로 비밀번호 등록

### Phase 1: 사용자 등록 시스템 완성

> 뼈대는 있고, 핵심 로직이 빠져있음

**CF Functions 구현 완성:**
- [ ] `register.js`
  - Gemini 파싱 프롬프트를 `LLM추출_프롬프트설계.md` 기반으로 교체 (지금은 대충 작성됨)
  - MySQL UPSERT 로직 실제 구현 (mysql2 라이브러리)
  - 이메일 발송 구현 (Resend API) — 등록 확인 메일 + edit_token 링크
- [ ] `send-magic-link.js`
  - MySQL에서 email로 edit_token 조회
  - 해당 이메일로 링크 재발송
- [ ] `profile/[token].js`
  - GET: token → 사용자 데이터 반환 (pre-fill용)
  - POST: 수정 내용 → Gemini 재파싱 → MySQL UPDATE

**users 테이블 추가 컬럼 필요:**

- [ ] `education_major` (TEXT) — 전공명

## 주요 기술 결정 사항 (변경 금지)

| 항목 | 결정 |
|------|------|
| 프론트엔드 | Cloudflare Pages (정적 HTML) |
| 백엔드 | Cloudflare Functions (Workers) |
| DB | MySQL 8 on 라즈베리파이 Docker |
| DB 연결 | Cloudflare Tunnel (`cloudflared`) |
| AI | Gemini API (스펙 파싱 + PDF 추출) |
| 이메일 | Resend API |
| 스크래퍼 | Python (라즈베리파이 Cron) |
| 결제 | 베타 기간 무료, 추후 토스페이먼츠 |
| 사용자 입력 | 자유서술 textarea (Gemini가 파싱) |

## 결제/수익화는 나중에

현재는 베타 테스트 기간으로 전체 무료.
토스페이먼츠 연동은 유효 사용자가 일정 수 이상 확보된 후 진행.
그 전까지는 관리자가 수동으로 `users.is_active`를 관리.
