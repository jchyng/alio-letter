# 사용자 플로우

> 기준일: 2026-03-13
> 시스템 플로우: [03_핵심기능_플로우.md](./03_핵심기능_플로우.md)

---

## 플로우 전체 지도

```
[A] 신규 가입      →  [B] 구독 신청  →  [D] 레포트 수신
                            ↑
[C] 로그인 & 수정  ─────────┘
```

| 플로우 | 진입점 | 핵심 조건 |
|--------|--------|-----------|
| A. 신규 가입 | 랜딩 → [무료로 시작하기] | 이름+이메일 필수 / 스펙·조건은 선택 |
| B. 구독 신청 | 마이페이지 [구독 시작] | 스펙·조건 둘 다 있어야 구독 가능 |
| C. 로그인 & 수정 | 랜딩 [내 정보 관리] 또는 이메일 링크 | 매직링크 방식 |
| D. 레포트 수신 | 시스템 자동 발송 | 구독 중(`is_active = 1`)인 사용자만 |

---

## FLOW A — 신규 가입

```
[랜딩 페이지]
    │
    ▼ [무료로 시작하기] 클릭
[회원등록 폼] /register
    │
    ├── 필수 입력
    │     이름, 이메일
    │
    └── 선택 입력
          나의 스펙 (자유서술)
          공고 조건 (자유서술)
    │
    ▼ [등록하기] 클릭
[클라이언트 유효성 검사]
    │  이름·이메일 빈 값 체크 / 이메일 형식 검사
    │  동의 체크박스 확인
    │
    ▼ 통과
[로딩] "등록 처리 중..."
    │
    ▼ POST /api/register
[CF Function]
    │  스펙·조건 텍스트가 있으면 → Gemini 파싱 → 구조화 저장
    │  없으면 → users 테이블만 생성 (스펙·조건 컬럼 NULL)
    │  확인 이메일 발송 (Resend API)
    │
    ▼
[성공 화면]
    "등록이 완료되었습니다!"
    "이메일로 확인 메일이 발송되었습니다."
    [구독 시작하기] ─→ FLOW B 진입
```

### 가입 완료 이메일

```
제목: [Alio-Letter] 등록이 완료되었습니다

안녕하세요, {이름}님!

등록이 완료되었습니다.

아직 구독을 시작하지 않으셨다면, 아래 링크에서
스펙·조건을 입력하고 레터 구독을 시작해보세요.

[내 정보 관리 바로가기]
→ https://alio-letter.pages.dev/profile/{edit_token}
```

## FLOW B — 구독 신청

> 가입 직후 또는 마이페이지에서 [구독 시작] 버튼을 눌렀을 때 진입.

```
[구독 시작] 버튼 클릭
    │
    ▼ 스펙·조건 보유 여부 확인
    │
    ├── 스펙 없음 또는 조건 없음
    │       ↓
    │   [안내 메시지]
    │   "레터 구독을 위해 스펙과 조건을 먼저 입력해주세요."
    │       ↓
    │   [스펙 입력 화면] (마이페이지 내 편집 모드)
    │   나의 스펙 (필수), 공고 조건 (필수)
    │       ↓ [저장하기]
    │   CF Function → Gemini 파싱 → MySQL UPDATE
    │       ↓
    │   구독 신청으로 자동 이동
    │
    └── 스펙·조건 모두 있음
            ↓
        [구독 확인 화면]
        "매일 새 공고가 분석되면 이메일로 알려드립니다."
        [구독 시작] 버튼
            ↓
        is_active = 1 업데이트
            ↓
        [구독 완료 메시지]
        "구독이 시작되었습니다! 공고가 매칭되면 이메일로 알려드릴게요."
```

### 구독 상태 정의

| 상태 | 조건 | 레포트 발송 |
|------|------|-------------|
| 미구독 | `is_active = 0` (가입만 한 상태) | X |
| 구독 중 | `is_active = 1` | O |
| 구독 중단 | `is_active = 0` (구독 후 중단) | X |

## FLOW C — 로그인 & 스펙·조건 수정

> 매직링크 방식. 비밀번호 없음.

```
[진입 경로 1] 랜딩 페이지 → [내 정보 관리] 클릭
[진입 경로 2] 가입 완료 이메일 → [내 정보 관리 바로가기] 클릭 (토큰 포함)
[진입 경로 3] 레포트 이메일 → [내 정보 수정하기] 클릭 (토큰 포함)

──────────────────────────────────────────────────

진입 경로 1: 이메일로 매직링크 요청

    [내 정보 관리] /login
        │  이메일 입력
        ▼ [링크 발송] 클릭
    POST /api/send-magic-link
        │  DB에서 이메일로 edit_token 조회
        │  해당 이메일로 매직링크 발송
        ▼
    [안내 메시지]
    "이메일함을 확인해주세요."
    (이메일에 등록되어 있지 않아도 동일 메시지 — 보안)
        │
        ▼ 이메일 내 링크 클릭
    [마이페이지] /profile/{edit_token}

──────────────────────────────────────────────────

진입 경로 2·3: 이메일 링크로 직접 진입

    이메일 내 링크 클릭
        ▼
    [마이페이지] /profile/{edit_token}

──────────────────────────────────────────────────

[마이페이지] 진입 후

    ▼ GET /api/profile/{token}
    │
    ├── 토큰 유효
    │       ↓
    │   폼 pre-fill
    │   이름, (이메일 읽기전용), 나의 스펙 원문, 공고 조건 원문
    │   구독 상태 표시 (구독 중 / 미구독)
    │
    └── 토큰 무효
            ↓
        "유효하지 않은 링크입니다. 등록 이메일을 확인해주세요."

──────────────────────────────────────────────────

[수정 제출]

    내용 수정 후 [수정하기] 클릭
        ▼
    POST /api/profile/{token}
        │  CF Function → Gemini 재파싱 → MySQL UPDATE
        ▼
    [성공] "정보가 수정되었습니다. 다음 분석부터 반영됩니다."
    [실패] 에러 메시지 + 입력값 유지
```

### 마이페이지 구성 요소

| 영역 | 내용 |
|------|------|
| 기본정보 | 이름 (수정 가능), 이메일 (읽기전용) |
| 나의 스펙 | 자유서술 textarea (원문 pre-fill, Gemini 재파싱) |
| 공고 조건 | 자유서술 textarea (원문 pre-fill, Gemini 재파싱) |
| 구독 관리 | 구독 상태 표시 + [구독 시작] 또는 [구독 중단] 버튼 |
| 알림 설정 | 이메일 알림 On/Off 토글 |
| 탈퇴 | [알림 수신 중단 및 데이터 삭제] |

## FLOW D — 레포트 수신 (시스템 → 사용자)

> 사용자가 직접 하는 행동은 없음. 시스템이 자동 발송.

```
[매일 오전 7시]
시스템이 구독 중인 사용자에게 분석 레포트 이메일 발송
    │  (FLOW 1~3 완료 기준)
    │
    ▼
[사용자] 이메일 수신
    │
    ├── 관심 있음
    │       ↓
    │   [잡알리오 원문 보기] 링크 클릭 → 잡알리오 상세 페이지
    │   [지원 바로가기] 링크 클릭 → 기관 채용 지원 페이지
    │
    ├── 스펙·조건 수정 필요
    │       ↓
    │   [내 정보 수정하기] 링크 클릭 → FLOW C 진입
    │
    └── 알림 끄고 싶음
            ↓
        [알림 해지] 링크 클릭 → is_active = 0 처리
```

### 레포트 이메일 구성 요약

```
┌─────────────────────────────────────────┐
│  [기관명] 채용공고 분석 리포트          │
│  마감: YYYY-MM-DD                       │
├─────────────────────────────────────────┤
│  🎯 매칭된 포지션                       │
│  채용유형 / 직무 / 인원 / 근무지        │
├─────────────────────────────────────────┤
│  ✅ 자격 요건 체크                      │
│  학력 ✅ / 병역 ✅ / 어학 ✅ ...        │
│  (미충족 항목은 ❌ 표시 + 이유)         │
├─────────────────────────────────────────┤
│  📊 서류 점수 예상                      │
│  어학: 토익 820점 → 48.2 / 50점        │
│  자격증: 전기기사 → 10점               │
│  예상 합계: 58.2 / 100점               │
├─────────────────────────────────────────┤
│  🏷️ 적용 가능 가점                     │
│  • 등록장애인: 필기 배점의 10%          │
├─────────────────────────────────────────┤
│  ⚠️ 직접 확인 필요                     │
│  • [기업 고유 가점 항목명]              │
├─────────────────────────────────────────┤
│  [잡알리오 원문 보기] [지원 바로가기]   │
│  [내 정보 수정하기] · [알림 해지]       │
└─────────────────────────────────────────┘
```

## 엣지 케이스 정리

| 상황 | 처리 방식 |
|------|-----------|
| 이미 가입된 이메일로 재등록 | UPSERT — 데이터 업데이트 + "기존 정보가 업데이트됩니다" 안내 |
| 스펙·조건 없이 구독 시도 | 구독 화면 진입 차단 → 입력 폼으로 안내 |
| 매직링크 만료 또는 무효 | "유효하지 않은 링크" + /login 이동 안내 |
| Gemini 파싱 실패 | 원문 텍스트는 저장 + "일부 항목을 인식하지 못했어요. 마이페이지에서 확인해주세요." 안내 |
| 구독 중 스펙·조건 수정 | 즉시 반영 — 다음 분석 사이클부터 적용 |
| 구독 중단 후 재구독 | 스펙·조건 있으면 즉시 재활성화 (`is_active = 1`) |
