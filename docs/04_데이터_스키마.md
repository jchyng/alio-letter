> DB 엔진: Cloudflare D1 (SQLite). 스키마 원본: `alio-letter-web/schema.sql`

---

## 테이블 관계 요약

```
postings (1)
  └── posting_tracks (1:N)   — Gemini 파싱 결과 (트랙별)

users (1)
  └── user_judgments (1:N)   — 트랙별 자격·가점 판정 결과

user_judgments (N)
  └── posting_tracks (FK)    — 어떤 트랙에 대한 판정인지
```

## 공고

### `postings` 테이블

잡알리오 목록/상세 페이지에서 크롤링한 공고 메타데이터.

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| posting_id | INTEGER PRIMARY KEY | 자동 증가 |
| alio_id | TEXT UNIQUE | 잡알리오 공고 고유번호 |
| title | TEXT | 공고 제목 |
| org_name | TEXT | 기관명 |
| posting_url | TEXT | 잡알리오 상세페이지 URL |
| deadline | TEXT | 접수 마감일 (ISO 8601) |
| registered | TEXT | 공고 등록일 |
| ncs | TEXT | 표준직무(NCS) |
| work_field | TEXT | 근무분야 |
| employment_type | TEXT | 고용형태 |
| location | TEXT | 근무지역 |
| education | TEXT | 학력정보 |
| recruit_type | TEXT | 채용구분 |
| is_substitute | TEXT | 대체인력 여부 |
| salary_url | TEXT | 연봉보러가기 URL |
| preferred | TEXT | 우대사항 원문 |
| attachment_path | TEXT | 첨부파일 로컬 경로 |
| attachment_ext | TEXT | 첨부파일 확장자 |
| attachment_converted | TEXT | PDF 변환 경로 |
| bonus_points | TEXT | 가산점 원문 |
| notes | TEXT | Gemini 분석 메모 (특정 필드로 분류하기 어려운 정보) |

### `posting_tracks` 테이블

Gemini가 공고문 PDF/HWP에서 추출한 트랙별 구조화 데이터.
1개 공고 = N개 트랙. (예: 대졸일반·대졸장애·고졸 = 3트랙)

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | INTEGER PRIMARY KEY | 자동 증가 |
| posting_id | INTEGER FK | postings.posting_id 참조 |
| track_name | TEXT | 채용구분명 (예: "대졸수준-일반") |
| positions | TEXT | 모집분야별 인원 원문 (예: "사무 3, 전기 5") |
| total_positions | INTEGER | 해당 트랙 총 채용인원 |
| eligibility | TEXT | 자격요건 JSON (`Eligibility` 참조) |

#### `eligibility` JSON 구조 (`Eligibility` TypedDict)

```json
{
  "education": "제한 없음",
  "career": "신입",
  "age": "제한 없음 (만 60세 미만)",
  "language": "제한 없음",
  "certificate": "제한 없음",
  "etc": "병역의무 불이행자 해당하지 않는 자"
}
```

## 사용자

### `users` 테이블

Cloudflare Functions을 통해 웹에서 등록한 사용자.

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | INTEGER PRIMARY KEY | 자동 증가 |
| email | TEXT UNIQUE | 이메일 (계정 식별 키 + 알림 발송 주소) |
| name | TEXT | 이름 |
| raw_spec_text | TEXT | 사용자가 입력한 원문 스펙 텍스트 |
| parsed_spec | TEXT | Gemini 파싱 결과 JSON (`UserProfile` 참조) |
| filter_prefs | TEXT | 공고 필터 조건 JSON (`PostingFilter` 참조) |
| edit_token | TEXT UNIQUE | 마이페이지 접근용 UUID (매직링크) |
| created_at | TEXT | 가입일, 기본값 `datetime('now')` |

#### `parsed_spec` JSON 구조 (`UserProfile` TypedDict)

```json
{
  "education": "4년제 대졸",
  "career_years": 0,
  "career_fields": [{"field": "전기", "years": 0}],
  "birth_year": 1999,
  "languages": [{"name": "TOEIC", "score": 820}],
  "certificates": ["전기기사", "소방설비기사"],
  "military": "필필",
  "veteran_type": "해당없음",
  "disability_grade": "해당없음",
  "is_low_income": false,
  "is_north_korean_defector": false,
  "is_independent_youth": false,
  "is_multicultural_child": false
}
```

#### `filter_prefs` JSON 구조 (`PostingFilter` TypedDict)

```json
{
  "ncs": ["사무원", "IT개발"],
  "locations": ["서울특별시", "경기도"],
  "work_fields": ["전산직", "사무직"],
  "employment_types": ["정규직", "무기계약직"],
  "recruit_types": ["신입", "신입+경력"],
  "educations": ["대졸(4년)", "석사"],
  "education_mode": "OR",
  "is_substitute": "전체",
  "org_types": ["공기업"],
  "org_names": ["한국전력공사"]
}
```

> 빈 배열 또는 누락된 키는 해당 항목 "전체"로 간주.

## 판정 결과

### `user_judgments` 테이블

사용자 × 공고 트랙 조합의 자격·가점 판정 결과.
동일 조합은 UNIQUE 제약으로 재판정 시 덮어씀.

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | INTEGER PRIMARY KEY | 자동 증가 |
| user_id | INTEGER FK | users.id 참조 |
| posting_track_id | INTEGER FK | posting_tracks.id 참조 |
| eligible | INTEGER | 자격요건 충족 여부 (1: 충족, 0: 미충족) |
| unmet | TEXT | 미충족 항목 목록 JSON (충족 시 `[]`) |
| bonus_summary | TEXT | 가산점 합계 요약 (예: "10%", "5점") |
| bonus_reasons | TEXT | 가산점 항목별 근거 JSON 배열 |
| judged_at | TEXT | 판정 수행 시각, 기본값 `datetime('now')` |
