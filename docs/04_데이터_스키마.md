# 데이터 스키마

> 핵심 기능 플로우: [03_핵심기능_플로우.md](./03_핵심기능_플로우.md)
> DB 엔진: Cloudflare D1 (SQLite). MySQL 전용 문법(`AUTO_INCREMENT`, `ENUM`, `ENGINE=InnoDB` 등) 미지원 — SQLite 문법으로 작성.

### MySQL → SQLite 타입 대응표

| MySQL | SQLite (D1) | 비고 |
|-------|-------------|------|
| `INT AUTO_INCREMENT PRIMARY KEY` | `INTEGER PRIMARY KEY` | SQLite에서 자동 증가 |
| `TINYINT(1)` | `INTEGER` | 0/1 값 동일하게 사용 |
| `ENUM('a','b')` | `TEXT CHECK (col IN ('a','b'))` | |
| `VARCHAR(n)` | `TEXT` | SQLite는 길이 제한 없음 |
| `JSON` | `TEXT` | D1에서 JSON 함수(`json_extract` 등) 사용 가능 |
| `DECIMAL(5,2)` | `REAL` | |
| `DATETIME` | `TEXT` | ISO 8601 형식: `'2026-03-14T06:00:00'` |
| `ON UPDATE CURRENT_TIMESTAMP` | 트리거로 대체 또는 앱 레이어 처리 | SQLite 미지원 |

<br>

## 테이블 관계 요약

```
users (1)
  ├── user_preferences (1:1)   — 관심 조건 필터
  ├── user_specs       (1:1)   — 본인 스펙
  ├── user_certificates (1:N)  — 보유 자격증 (자유 텍스트)
  └── user_posting_analyses (1:N) — 공고 분석 결과

postings (1)
  ├── posting_tracks (1:N)     — 채용 트랙별 파싱 결과
  └── user_posting_analyses (1:N)
```

### `user_specs` 테이블 — 본인 스펙

> 자격 체크 + 가점 계산의 입력 데이터.
> 자격증만 1:N이므로 `user_certificates`로 분리.

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| user_id | INTEGER PRIMARY KEY | PK/FK | users.user_id 참조 |
| birth_date | TEXT | O | 생년월일 (ISO 8601, 나이 제한 체크용) |
| gender | TEXT | O | 성별 ('남'/'여', 양성평등 채용목표제 체크용) |
| education_level | TEXT | O | 최종학력 (고졸, 전문대졸, 대졸(4년), 석사, 박사) |
| education_status | TEXT | O | 졸업상태 (졸업, 졸업예정, 재학, 중퇴) |
| major_category | TEXT | - | 전공계열 (이공계, 상경계, 인문사회계, 예체능계, 기타) |
| school_name | TEXT | - | 학교명 |
| school_region | TEXT | - | 학교 소재지 (시/도 단위) — 이전지역인재 채용목표제 체크용 |
| military_status | TEXT | O | 병역구분 (군필, 미필, 면제, 현역, 해당없음) |
| military_discharge_date | TEXT | - | 전역(예정)일 (ISO 8601, 현역일 때만) |
| is_disabled | INTEGER | O | 장애인 여부, 기본값 0 |
| disability_type | TEXT | - | 장애유형 (is_disabled=1일 때) |
| is_veteran | INTEGER | O | 보훈대상자 여부, 기본값 0 |
| veteran_type | TEXT | - | 보훈유형 (is_veteran=1일 때) |
| residence_region | TEXT | - | 거주지역 (시/도 단위) |
| residence_detail | TEXT | - | 상세 거주지 (발전소주변지역 가점 체크용) |
| toeic_score | INTEGER | - | TOEIC 점수 |
| toeic_expiry | TEXT | - | TOEIC 유효기간 만료일 (ISO 8601) |
| toeic_speaking_score | INTEGER | - | TOEIC Speaking 점수 |
| toeic_speaking_expiry | TEXT | - | TOEIC Speaking 유효기간 만료일 (ISO 8601) |
| opic_grade | TEXT | - | OPIc 등급 (AL, IH, IM3, IM2, IM1, IL) |
| opic_expiry | TEXT | - | OPIc 유효기간 만료일 (ISO 8601) |
| is_low_income | INTEGER | O | 저소득층 여부, 기본값 0 |
| is_north_korean_defector | INTEGER | O | 북한이탈주민 여부, 기본값 0 |
| is_multicultural_family | INTEGER | O | 다문화가족 여부, 기본값 0 |
| is_independent_youth | INTEGER | O | 자립준비청년 여부, 기본값 0 |
| is_career_break_woman | INTEGER | O | 경력단절여성 여부, 기본값 0 |
| is_part_time_worker | INTEGER | O | 시간선택제근로자 여부, 기본값 0 |

> **boolean 가점 필드 설계 이유**: 위 항목들은 정부 정책 기반 가점 카테고리로, 다수의 공기업 공고에서 공통적으로 등장한다. 사용자 스펙은 고정값이고, 매칭 시점에 해당 공고의 `bonus_points` 목록과 대조한다.
>
> **기업 고유 가점 항목 처리**: 위 목록에 없는 항목(예: 특정 기업 인턴 이수 여부)은 지원되지 않으며, 이메일에 "직접 확인 필요" 항목으로 안내한다.
>
> **어학성적 유효기간**: 공고 마감일 기준으로 만료 여부 자동 체크. 여러 시험 성적이 있으면 `language_conversion_table`로 TOEIC 환산 후 최고점 적용.

### `user_certificates` 테이블 — 보유 자격증

> 사용자가 자유 텍스트로 입력한 자격증 정보를 저장한다. 공식 자격증명 정규화·분야 매핑·점수 계산은 AI가 담당하므로 마스터 테이블 참조가 불필요하다.

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| cert_id | INTEGER PRIMARY KEY | PK | 자동 증가 |
| user_id | INTEGER | FK | users.user_id 참조 |
| cert_name | TEXT | O | 자격증명 자유 텍스트 (예: 전기기사, 정보처리기사) |
| issue_date | TEXT | O | 취득일 (ISO 8601) |

> Gemini가 사용자의 자유서술 스펙 텍스트에서 자격증명과 취득일을 추출해 삽입한다.

**가점 계산 흐름**:
1. `user_certificates` 자유 텍스트 목록 + `certificate_bonus_table` + 지원 `job_field` → Gemini API 호출
2. Gemini가 분야별·공통 자격증 매칭, 동일종류 상위등급 1개만 인정, 최대 3개 합산 규칙 적용
3. 매칭 결과 + 점수 + 근거 JSON → `user_posting_analyses.certificate_score_detail`에 저장

## 분석 결과

### `user_posting_analyses` 테이블

> 사용자 × 공고 트랙 조합의 분석 결과 저장 + 이메일 발송 이력 관리.

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| analysis_id | INTEGER PRIMARY KEY | PK | 자동 증가 |
| user_id | INTEGER | FK | users.user_id 참조 |
| posting_id | INTEGER | FK | postings.posting_id 참조 |
| track_id | INTEGER | FK | posting_tracks.track_id 참조 |
| is_eligible | INTEGER | O | 지원 가능 여부 (1: 가능, 0: 불가) |
| ineligible_reasons | TEXT | - | 미충족 항목 목록 (JSON, is_eligible=0일 때) |
| language_score | REAL | - | 서류 어학 예상 점수 |
| certificate_score | REAL | - | 서류 자격증 예상 점수 |
| total_document_score | REAL | - | 서류 합계 예상 점수 |
| certificate_score_detail | TEXT | - | AI 자격증 매칭 근거 (JSON: matched_field_certs, matched_common_certs, excluded_certs, has_advanced_cert, rules_applied) |
| applicable_bonus_points | TEXT | - | 적용 가능 가점 목록 및 배점률 (JSON) |
| unchecked_bonus_points | TEXT | - | 시스템이 판단 불가하여 사용자가 직접 확인해야 할 가점 항목 (JSON) |
| quota_policy_matches | TEXT | - | 채용목표제 해당 여부 (JSON) |
| analyzed_at | TEXT | O | 분석 수행 시각, 기본값 CURRENT_TIMESTAMP |
| email_sent_at | TEXT | - | 이메일 발송 시각 (NULL이면 미발송, ISO 8601) |
