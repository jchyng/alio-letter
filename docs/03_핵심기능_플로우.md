# 핵심 로직 플로우

## 한 줄 요약

잡알리오 채용공고 파일을 AI가 읽어 핵심 정보를 구조화하고 요약 → 사용자 스펙과 자동 매칭 → 지원 가능 여부와 가점 체크리스트까지 제공

## 핵심 데이터

### 1. 사용자 관심 조건 (필터)

> 사용자가 등록 시 입력하는 관심 조건.
> **잡알리오 검색 필터와 동일한 분류 체계**를 사용하여, 공고 메타데이터와 직접 매칭한다.
> 빈 값(NULL/빈 배열)은 "전체"로 간주하여 해당 항목 필터링을 건너뛴다.

#### `users` 테이블

| 컬럼명                  | 타입                          | 필수 | 설명                                     |
| ----------------------- | ----------------------------- | ---- | ---------------------------------------- |
| user_id                 | INT                           | PK   | AUTO_INCREMENT                           |
| name                    | VARCHAR(50)                   | O    | 이름                                     |
| email                   | VARCHAR(100)                  | O    | 이메일 (알림 발송 주소)                  |
| is_active               | TINYINT(1)                    | O    | 활성 상태 (1: 활성, 0: 비활성), 기본값 1 |
| subscription_status     | ENUM('free','paid','expired') | O    | 구독 상태, 기본값 'free'                 |
| subscription_expires_at | DATE                          | -    | 구독 만료일 (paid일 때만)                |
| created_at              | DATETIME                      | O    | 가입일, 기본값 NOW()                     |
| updated_at              | DATETIME                      | O    | 수정일, 기본값 NOW() ON UPDATE           |

#### `user_preferences` 테이블

| 컬럼명                | 타입 | 필수  | 설명                             |
| --------------------- | ---- | ----- | -------------------------------- |
| user_id               | INT  | PK/FK | users.user_id 참조               |
| pref_regions          | JSON | -     | 희망 근무지역 (복수선택)         |
| pref_job_fields       | JSON | -     | 희망 근무분야 (복수선택)         |

| pref_recruit_types    | JSON | -     | 희망 채용구분 (복수선택)         |
| pref_education_levels | JSON | -     | 학력정보 (복수선택)              |
| pref_org_types        | JSON | -     | 관심 기관유형 (복수선택)         |
| pref_org_names        | JSON | -     | 관심 기관명 (복수선택, 선택사항) |

#### 매칭 로직

```
공고 메타데이터의 각 항목이 사용자 필터 JSON 배열에 포함되면 매칭.
필터가 비어있으면(NULL 또는 빈 배열) → 해당 항목은 "전체"로 간주 → 무조건 매칭.

예시: 한국남부발전 공고 (부산광역시, 기술직, 정규직, 신입, 대졸(4년), 공기업)
→ 사용자A: pref_regions=["부산광역시","서울특별시"], pref_job_fields=["기술직"] → 매칭 O
→ 사용자B: pref_regions=["서울특별시"], pref_job_fields=["전산직"] → 매칭 X (지역, 분야 모두 불일치)
→ 사용자C: pref_regions=[], pref_job_fields=["기술직"] → 매칭 O (지역 전체 + 분야 일치)
```

## 핵심 기능

### 공고 데이터 수집 플로우

1. DB에서 가장 마지막으로 등록된 공고 번호(last_no) 조회
2. 잡알리오 사이트(https://job.alio.go.kr/recruit.do?ing=2)에서 가장 최신 공고부터 last_no까지 각 공고의 메타데이터와 상세페이지 링크 정보 Stack에 저장
3. Stack에서 하나씩 뽑으면서 해당 공고 상세페이지로 이동.
4. 상세페이지에서 [해당 공고 URL, 공고 제목, 공고문 파일]을 수집한다.
5. 수집한 공고문 파일[여러 이미지형식, hwp, hwpx, pdf]를 gemini API로 분석해서 필요한 정보[]를 추출한다.
6. []을 DB에 저장한다.

### 공고 자동 분석 및 결과 알림

1. 새로 수집한 공고문의 메타데이터로 DB의 유저 중에서 해당 공고에 관심[원하는 지역, 기업, 기관 유형]이 있는 유저를 조회
2. 없다면 넘어가고, 1명이라도 있다면 각 사용자의 스펙 정보[학력, 자격증, 어학성적, 보훈 여부, 장애 여부]를 토대로 gemini API를 활용해서 공고에 대한 자격 조건 체크와 가점 계산 수행.
3. 수행된 결과를 DB에 각 사용자의 해당 공고별 요약 내용 테이블에 저장
4. 1~3을 새로운 공고가 없을 때까지 반복
5. 사용자의 공고별 요약 내용 중, 발송된 적 없는 내용을 사용자 이메일로 발송
6. 해당 공고는 발송된 상태로 변경
