> 기준일: 2026-03-13
> 아래 결정 사항은 변경 금지. 변경 필요 시 팀 합의 필요.

## 기술 스택

| 구분 | 선택 | 비고 |
|------|------|------|
| 프론트엔드 | Cloudflare Pages (정적 HTML) | Pretendard 폰트, 모바일 퍼스트 |
| 백엔드 API | Cloudflare Functions (Workers) | 서버리스, Edge 기반 |
| DB 연결 | Cloudflare Tunnel (`cloudflared`) | 포트 개방 없이 라즈베리파이 MySQL 직접 연결 |
| 데이터베이스 | MySQL 8 (라즈베리파이 Docker) | 관계형 데이터 처리 |
| AI 파싱 | Gemini 1.5 Flash API | 스펙 파싱 + 공고 PDF 추출 |
| 이메일 발송 | Resend API | 확인 메일 + 분석 레포트 |
| 배치 오케스트레이션 | n8n (라즈베리파이 Docker) | 공고 수집 → 매칭 → 발송 자동화 |
| 공고 스크래퍼 | Python (라즈베리파이) | n8n Execute Command 노드에서 호출 |
| 결제 | 베타 기간 무료 → 추후 토스페이먼츠 | 현재 미구현 |

## 시스템 아키텍처

```
[사용자 브라우저]
    │
    ├── 정적 페이지 요청
    │       ↓
    │   [Cloudflare Pages]
    │   /  /register  /login  /profile/:token
    │
    └── API 요청
            ↓
        [Cloudflare Functions (Edge)]  ─────────────────────────────┐
            │                                                        │
            ├── Gemini API 호출 (스펙/조건 파싱)                    │
            ├── Resend API 호출 (가입 확인 메일)                    │
            └── Cloudflare Tunnel → MySQL 쿼리 (UPSERT/SELECT)      │
                                                                     │
─────────────────────────────────────────────────────────────────── │
                                                                     │
[라즈베리파이 (Home Network)]  ←─────────────────────────────────── ┘
    │
    ├── cloudflared (Tunnel Daemon)
    │     Cloudflare와 아웃바운드 터널 유지 / 포트 외부 개방 없음
    │
    ├── MySQL 8 (Docker)
    │     사용자 데이터 + 공고 데이터 + 분석 결과 저장
    │
    ├── n8n (Docker)
    │     ├── 워크플로우 1: 공고 수집 파이프라인 (매일 오전 6시)
    │     │     Python scraper.py 실행
    │     │     → Gemini API PDF 파싱
    │     │     → MySQL 저장
    │     │     → 워크플로우 2 트리거
    │     │
    │     └── 워크플로우 2: 매칭 + 이메일 발송 (워크플로우 1 완료 직후)
    │           MySQL 매칭 쿼리
    │           → 자격 체크 + 가점 계산 (Code 노드)
    │           → Resend API 이메일 발송 (오전 7시)
    │
    └── Python Scraper
          잡알리오 공고 목록 수집 / 신규 공고 감지 / PDF 다운로드
```

## 역할 분리 원칙

| 처리 유형 | 담당 | 이유 |
|-----------|------|------|
| 실시간 (사용자 요청) | Cloudflare Functions | 빠른 응답 필요 |
| 배치 (공고 수집·발송) | n8n + Python (라즈베리파이) | 장시간 실행, CF Functions 제한 회피 |
| AI 파싱 (스펙) | CF Functions → Gemini | 사용자 요청 흐름 내 동기 처리 |
| AI 파싱 (공고 PDF) | n8n → Gemini | 배치 흐름 내 비동기 처리 |
| 자격 체크·점수 계산 | n8n Code 노드 (코드 직접 계산) | 정확성·재현성 보장 (LLM 미사용) |

## 보안 연결 구조 (Cloudflare Tunnel)

```
[Cloudflare Functions]
    │  mysql2 라이브러리
    │  접속 주소: db.internal.example.com (Tunnel 도메인)
    ▼
[Cloudflare Zero Trust]
    │  Worker IP만 접근 허용 (Access Policy)
    ▼
[cloudflared on 라즈베리파이]
    │  아웃바운드 전용 터널
    ▼
[MySQL Docker :3306]
```

- 라즈베리파이 포트를 외부에 개방하지 않음
- Cloudflare Worker 이외의 출처는 Tunnel 도메인 접근 불가

## 실행 스케줄 (n8n)

| 워크플로우 | 트리거 | 내용 |
|------------|--------|------|
| 공고 수집 | 매일 오전 6시 | 잡알리오 신규 공고 크롤링 + Gemini PDF 파싱 + MySQL 저장 |
| 매칭·분석 | 수집 완료 직후 자동 | 신규 공고 × 구독 사용자 매칭 + 자격 체크 + 점수 계산 |
| 이메일 발송 | 매일 오전 7시 | 분석 완료된 결과 일괄 발송 (Resend API) |
| 마감 공고 처리 | 매일 오전 0시 | `deadline < NOW()` → `status = '마감'` |
