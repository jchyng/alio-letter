# 구체적 구현 계획 (v2.1 - 통합 아키텍처)

## 기술 스택

| 구분             | 선택                              | 이유                                                    |
| ---------------- | --------------------------------- | ------------------------------------------------------- |
| **백엔드/API**   | Cloudflare Functions (Workers)    | 서버리스 아키텍처, Edge 기반 빠른 응답, 저비용          |
| **보안 연결**    | Cloudflare Tunnel (`cloudflared`) | 포트 개방 없이 라즈베리파이 DB에 안전하게 직접 연결     |
| **데이터베이스** | MySQL (라즈베리파이 Docker)        | 기존 데이터 유지, 관계형 데이터 처리에 적합             |
| **PDF 파싱**     | Gemini 1.5 Flash API              | Cloudflare Functions에서 직접 호출 (구조화 추출 최적화) |
| **배치 오케스트레이션** | **n8n (라즈베리파이 Docker)** | 공고 수집 → 매칭 → 이메일 발송 파이프라인 자동화. 스케줄링, 에러 처리, 재시도, 워크플로우 시각화 제공 |
| **스크래핑**     | Python (라즈베리파이)              | n8n Execute Command 노드에서 호출하는 공고 수집 스크립트 |
| **알림 발송**    | Resend API 또는 Gmail API         | n8n Gmail/HTTP 노드에서 이메일 발송                     |
| **결제**         | **베타 테스트 (무료)**             | 베타 기간 동안 전체 기능 무료 제공 및 사용자 피드백 확보 |

## 베타 테스트 운영 전략

1. **전체 기능 무료:** 기본/프리미엄 구분 없이 모든 분석 및 알림 기능을 무료로 개방합니다.
2. **사용자 검증:** 실제 사용자가 얼마나 공고 분석 리포트에 만족하는지, 이메일 오픈율은 어떤지 지표를 수집합니다.
3. **수익화 시점:** 유효 사용자(DAU/WAU)가 일정 수준 이상 확보되고 기능 안정성이 검증된 후 토스페이먼츠 연동을 검토합니다.

## 시스템 아키텍처

```
[사용자 브라우저]
  ├── 정적 웹 (Cloudflare Pages)
  └── API 요청 (Cloudflare Functions)

[Cloudflare Functions (Edge)]  ← 실시간 처리
  ├── 입력 검증 & Gemini API 호출 (스펙 파싱)
  ├── [Cloudflare Tunnel]을 통한 MySQL 직접 쿼리 (UPSERT/SELECT)
  └── 이메일 API 호출 (회원가입 확인 메일)

[라즈베리파이 (Home Network)]  ← 배치 처리
  ├── cloudflared (Tunnel Daemon): 외부 포트 개방 없음
  ├── MySQL Docker Container: 사용자/공고 데이터 저장
  ├── n8n Docker Container: 배치 워크플로우 오케스트레이션
  │     ├── 워크플로우 1: 공고 수집 파이프라인 (매일 2회 Schedule)
  │     │     Python 스크립트 실행 → Gemini PDF 분석 → MySQL 저장
  │     └── 워크플로우 2: 매칭 + 이메일 발송 (공고 저장 후 트리거)
  │           MySQL 매칭 쿼리 → 리포트 HTML 생성 → 이메일 발송
  └── Python Scraper: n8n이 호출하는 잡알리오 공고 수집 스크립트
```

---

## 핵심 구현 전략: Cloudflare Tunnel DB 연결

1. **보안 연결:** 라즈베리파이에서 `cloudflared`를 실행하여 Cloudflare와 아웃바운드 터널을 맺습니다.
2. **Access 제어:** Cloudflare Zero Trust 설정을 통해 오직 본인의 Cloudflare Worker에서 오는 요청만 DB 터널 도메인(예: `db.internal.example.com`)에 접근할 수 있도록 격리합니다.
3. **직접 쿼리:** Cloudflare Functions에서 `mysql2` 라이브러리를 사용하여 터널 주소를 통해 MySQL에 직접 연결 및 쿼리를 실행합니다.

## 워크플로우 상세 설계

### 1. 회원가입 및 프로필 관리 (Real-time)

**트리거:** 사용자 폼 제출 (POST /api/register)

1. **입력 검증:** 이름, 이메일, 스펙 텍스트 유효성 검사.
2. **스펙 파싱 (Gemini):** 사용자 입력 텍스트를 구조화된 JSON(학력, 자격증 등)으로 변환.
3. **DB 저장 (MySQL):** Cloudflare Tunnel을 통해 라즈베리파이 DB에 사용자 정보 저장.
4. **확인 메일:** 가입 완료 및 프로필 수정 전용 토큰 링크 발송.

### 2. 공고 수집 파이프라인 (n8n 워크플로우 1)

**트리거:** n8n Schedule 노드 (매일 오전 9시, 오후 3시)

```
[n8n Schedule Trigger]
    ↓
[Execute Command 노드] Python scraper.py 실행
    → 잡알리오 신규 공고 목록 수집
    → 이미 수집된 공고 중복 필터링 (MySQL 조회)
    → 신규 공고 PDF 다운로드
    ↓
[HTTP Request 노드] Gemini API 호출
    → PDF → 구조화 JSON 추출 (LLM추출_프롬프트설계.md 프롬프트 사용)
    → headcount 합계 검증
    ↓
[MySQL 노드] 공고 데이터 저장
    → institutions → job_postings → recruitment_sections
    → recruitment_units → qualification_requirements
    → selection_stages → bonus_points → recruitment_targets
    ↓
[워크플로우 2 트리거] 신규 공고 ID 전달
```

### 3. 매칭 + 이메일 발송 (n8n 워크플로우 2)

**트리거:** 워크플로우 1 완료 후 자동 호출 (또는 Webhook 노드)

```
[Webhook / 워크플로우 트리거] 신규 공고 ID 수신
    ↓
[MySQL 노드] 매칭 쿼리 실행
    → 공고 조건(직무/지역/채용구분)에 맞는 활성 사용자 조회
    → (DB설계.md 매칭 쿼리 참고)
    ↓
[Code 노드] 사용자별 리포트 데이터 생성
    → 자격 요건 ✅/❌ 체크
    → 서류 점수 예상 계산 (토익 환산 등)
    → 가점 항목 필터링
    ↓
[Gmail / HTTP Request 노드] 이메일 발송
    → 사용자별 개인화 HTML 리포트 전송
    → (UI설계.md 분석 리포트 이메일 템플릿 참고)
```
